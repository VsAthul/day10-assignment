# BankDoc AI — RAG-Powered PDF Chat Application

A Retrieval-Augmented Generation (RAG) web application that lets you ask questions about a banking PDF document through a conversational chat interface.

---

## Project Overview

BankDoc AI displays a banking PDF on the left side of the screen and provides an AI-powered chat interface on the right. When you ask a question, the app embeds your query, retrieves the most relevant text chunks from the document (stored in ChromaDB), and passes them as context to a local Ollama LLM to generate an accurate, grounded answer. All conversations are persisted in a SQLite database.

---

## How It Works (RAG Flow)

```
User Question
     │
     ▼
Embed query using qwen3-embedding:0.6b (Ollama)
     │
     ▼
Search ChromaDB → Top 5 most relevant document chunks
     │
     ▼
Build prompt: [System instruction + Context chunks + Question]
     │
     ▼
Send to Ollama LLM (qwen3: 0.6b) → Generate answer
     │
     ▼
Save conversation to SQLite → Return answer to UI
```

**Ingestion (run once before the app):**
```
PDF file → pypdf extracts text → chunk into 500-char pieces (100-char overlap)
        → embed each chunk (qwen3-embedding:0.6b) → store in ChromaDB
```

## Setup Instructions

### 1. Clone the Repository

```bash
git clone https://github.com/<your-username>/day10-assignment.git
cd day10-assignment
```

### 2. Create a Virtual Environment

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
```

### 3. Install Python Dependencies

```bash
pip install -r requirements.txt
```

### 4. Install and Start Ollama

Download Ollama from [https://ollama.com](https://ollama.com) and install it.

Then pull the required models:

```bash
ollama pull qwen3-embedding:0.6b
ollama pull llama3.2:latest
```

Start the Ollama server (if not already running):

```bash
ollama serve
```

### 5. Add the Banking PDF

Download a banking PDF and save it to:

```
./static/document.pdf
```

---

## How to Run

### Step 1 — Ingest the PDF (run once)

```bash
python ingest.py
```

This reads the PDF, creates chunks, generates embeddings, and saves everything to ChromaDB in `./my_chroma_db/`.

### Step 2 — Start the Flask App

```bash
python app.py
```

Then open your browser at: [http://localhost:5000](http://localhost:5000)

## Database Schema

Conversations are stored in `instance/conversations.db` (SQLite):

| Column        | Type     | Constraints         |
|---------------|----------|---------------------|
| id            | Integer  | Primary Key, Not Null |
| user_message  | Text     | Not Null            |
| ai_response   | Text     | Not Null            |
| created_at    | DateTime | Not Null            |
